* DBCP(DataBase Connection Pool)
  - 데이터베이스 연동 작업 중 시간이 가장 많이 소요되는 작업은 connection(접속) 수립과 해제.
  - 서버에 미리 접속을 수립한 객체를 여러개 만들어서 필요시 대여해주고 돌려받는 방식 활용
    접속 수립과 해제에 드는 시간을 감소시키고 전체 서비스 제공을 원할하게 처리하는 방식


* 비밀번호 암호화
  - 사용자가 입력한 비밀번호는 그대로(plain text) 저장하지 않고 보안을 위해 암호화(cypher text) 하여 DB에 저장
  - Spring Security 에서 제공하는 비밀번호 암호화 객체를 사용
  - BCryptPassWordEncoder class : 두가지 메소드를 제공
    1) encode("plain") -> 입력한 문자열을 암호화 (회원가입 시)
    2) matches("plain" , "cypher") -> 두 값을 비교 (로그인 시)
  - 회원가입 -> 원문 비밀번호 -> 암호화 -> Dto 에 재저장 -> insert


* Session 기반 로그인 유지
  - 세션(Session)이란 웹 사이트에서 여러 페이지에 걸쳐 사용되는 데이터 저장 방법
  - 클라이언트의 브라우저에서 해당 서버에 접속하면 세션이 생성됨 (1인당 1개)
  - 접속을 종료하거나 브라우저를 종료하면 세션은 제거됨
    (일정 시간을 유지할 수 있음, 기본 유효시간 30분, spring boot 는 15분)
  - 유효시간은 설정할 수 있음
  - 인터넷 서비스의 특성 상 로그인 처리는 한 페이지에서만 유효함
    로그인 상태를 유지하려면 로그인 성공정보를 모든 페이지에서 유지해야 한다 -> 세션을 사용하는 이유


* 세션의 삭제(로그아웃)
  - 세션에 저장한 데이터만 삭제: session.removeAttribute("이름");
  - 세션 자체를 삭제: session.invalidate();
